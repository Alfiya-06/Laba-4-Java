### 2. Класс Line<T extends Point>
Обобщённый класс, хранящий начало и конец как объекты T, где T — тип, наследующийся от Point.(Кдассы `Point` и `Point3` взяты из предыдущих лаб)

Поддерживает как 2D, так и 3D точки.

Имеет конструктор с двумя параметрами.

Содержит методы для получения и установки начала и конца.

Может вычислять длину линии с учётом Z-координаты, если точки 3D.

Возвращает строковое представление: "Линия от ... до ...".

Представляет обобщённую линию между двумя точками типа `T`;

Тип `T` ограничен границей `Point`, что гарантирует наличие методов `getX()` и `getY()`;

**Примеры:**

`new Line<>(new Point(0, 0), new Point(3, 4)).getLength()` → 5.0

`new Line<>(new Point3(0, 0, 0), new Point3(3, 4, 5)).getLength()` → 7.0710678118654755

`new Line<>(new Point(1, 1), new Point(2, 2)).toString()` → "Линия от {1;1} до {2;2}"

### 4. Интерфейс `Filter<T>`

Функциональный интерфейс, определяющий условие фильтрации.

Содержит метод `boolean test(T value)`.

### 5. Класс `Filters`

Содержит статический обобщённый метод `filter`, который:

Принимает список и фильтр.

Возвращает новый список, содержащий только элементы, удовлетворяющие условию.

**Пример:**

`Filters.filter(Arrays.asList(1, -3, 5), n -> n > 0)` → [1, 5]

### 6. Интерфейс `Function<T, P>`
    
Функциональный интерфейс, определяющий преобразование одного типа в другой.

Содержит метод `P apply(T value)`.

### 7. Класс `Functions`

Содержит статический обобщённый метод transform, который:

Принимает список и функцию.

Возвращает новый список, в котором каждый элемент — результат применения функции.

**Пример:**

`Functions.transform(Arrays.asList("hello", "world"), String::length)` → [5, 5]
